# スキルツリー ベースラインデータ(IT系4職種)

出典: roadmap.sh(GitHubで36万スター超のコミュニティ標準ロードマップ)の
Frontend / Backend / AI Engineer / UX Design ロードマップをマイルストーン粒度に再構成したもの。
本ファイルの「プロンプト埋め込み用ブロック」を、ツリー生成プロンプトの
[ベースライン骨格] にそのまま貼り付けて使う。

**3階層モデルへの対応**: 本ファイルのM系列(M0〜M7)= **木(Trunk)** の候補、
各行の「|」区切り項目 = **枝(Branch)** の候補として使う。
**葉(Leaf)はこのファイルには定義しない**(get-or-generate-leavesがユーザーの
学習条件に合わせて動的生成する)。また、Step1の12職種のうち本ファイルに
ベースラインがない職種(モバイル等)は、最も近い職種のブロックを軸に
AIが調整する(例: モバイル→フロントエンド軸、データサイエンティスト→AI/データ系軸)。

主な参照元:
- https://roadmap.sh/frontend
- https://roadmap.sh/backend (https://roadmap.sh/pdfs/roadmaps/backend.pdf)
- https://roadmap.sh/ai-engineer (https://roadmap.sh/pdfs/roadmaps/ai-engineer.pdf)
- https://roadmap.sh/ux-design (https://roadmap.sh/pdfs/roadmaps/ux-design.pdf)

## 0. IT共通幹(全職種共有 / 目標変更時も習得済みを引き継ぐ)

- M0 プログラミング基礎: 変数と型 / 制御構文 / 関数 / 基本データ構造(配列・連想配列) / オブジェクト指向の初歩 / エラーとデバッグ
- M1 開発の道具: Git / GitHub / コマンドライン基礎 / エディタと開発環境
- M2 Webの仕組み: インターネットとHTTP / ブラウザとサーバーの役割 / HTML基礎 / CSS基礎

※ UI/UXデザイナー志望の場合、M0は「変数と型 / 制御構文」までの軽量版とし、
   M2(HTML/CSS)を重点化する。

## 1. フロントエンドエンジニア(共通幹 M0-M2 の続き)

- M3 JavaScript実践: DOM操作 / 非同期処理(Promise, async/await) / ES6+の文法 / npmとパッケージ管理
- M4 フレームワーク: React(またはVue)の基礎 / コンポーネント設計 / 状態管理 / ルーティング
- M5 品質と体験: レスポンシブデザイン / アクセシビリティ / ブラウザ開発者ツール / テストとデバッグ / SEO基礎
- M6 実践と資産化: REST APIの利用 / 小さなWebアプリの制作 / パフォーマンス基礎 / ポートフォリオ制作

## 2. バックエンドエンジニア(共通幹 M0-M2 の続き)

- M3 サーバーサイド言語: 言語を1つ選び深める(Python / Node.js / Java 等) / 標準ライブラリ / パッケージ管理
- M4 データベース: リレーショナルDB(PostgreSQL等) / SQL基礎 / 正規化 / トランザクションとACID / ORM入門
- M5 API開発: RESTの原則 / JSON API設計 / 認証・認可の基礎 / Webセキュリティ基礎
- M6 運用の入り口: テスト(ユニット/統合) / CI/CD入門 / キャッシュの基礎 / クラウドへのデプロイ入門
- M7 実践と資産化: 小さなAPIサービスの制作(ToDo API・URL短縮など) / ポートフォリオ制作

## 3. AI/データ系エンジニア(共通幹 M0-M2 の続き / M0の言語はPython推奨)

- M3 Pythonとデータ処理: Python文法の深掘り / pandas等でのデータ処理 / Jupyter環境
- M4 機械学習の基礎: 統計の初歩 / 教師あり学習の概念 / モデル評価の考え方(過学習・精度指標)
- M5 LLM活用: 事前学習済みモデルの理解 / LLM API(Chat Completions等)の利用 / プロンプトエンジニアリング / トークンと料金の管理
- M6 応用: 埋め込み(Embeddings)とRAG / AIエージェントの基礎 / 出力評価(LLM-as-a-judge等) / AI安全と倫理
- M7 実践と資産化: LLMを使った小さなアプリ制作 / ポートフォリオ制作

## 4. UI/UXデザイナー(共通幹は軽量版 / M2重点)

- M3 デザインの基礎: 色 / タイポグラフィ / レイアウトと視覚原則
- M4 UXの考え方: 人間の意思決定の理解 / ユーザー調査の基礎 / ユーザーストーリー / カスタマージャーニーマップ
- M5 制作スキル: ワイヤーフレーム / Figma / プロトタイピング / UXパターン
- M6 検証と改善: ユーザビリティテスト / 行動変容とゲーミフィケーション / 実装者との協働(HTML/CSSの理解)
- M7 実践と資産化: ケーススタディ制作 / ポートフォリオ制作

## AIへの指示ルール(ベースラインと併せて必ずプロンプトに含める)

1. 上記の骨格に存在しないマイルストーンを新設しない
2. ノードの追加は「個人枝」として1マイルストーンにつき最大2〜3個まで
3. done は絶対に割り当てない(doneはクイズ合格後にサーバーだけが設定する)。
   経験タグ・深掘り回答に根拠があるノードは in_progress または unlocked にする
4. 各ノードに prerequisite_ids(依存するノードIDの配列)を必ず設定する。循環させない
5. 各ノードの how_to_learn は1〜2文で具体的に(教材名や作るものを示す)
6. 出力は指定JSONスキーマのみ

## 生成の期待値見本(プロンプト検証時の正解イメージ)

generate-tree / get-or-generate-leaves の出力がこの粒度に近いかを検証の基準にする。

### 目標「AI・機械学習エンジニア」の場合

```
木: プログラミング基礎
├─ 枝: Python基礎
│  ├─ 葉: 変数と型を確認する
│  ├─ 葉: if文を3問実装する
│  └─ 葉: リスト処理を練習する
└─ 枝: Git・開発環境

木: アルゴリズム
├─ 枝: 計算量 / 探索 / ソート

木: 機械学習
├─ 枝: データ前処理 / 教師あり学習 / モデル評価

木: 深層学習
├─ 枝: ニューラルネットワーク基礎 / CNN / Transformer
```

### 目標「UIデザイナー」の場合

```
木: デザイン基礎
├─ 枝: 色彩 / タイポグラフィ / レイアウト

木: UI設計
├─ 枝: コンポーネント設計 / デザインシステム / レスポンシブデザイン

木: UX設計
├─ 枝: ユーザー調査 / 情報設計 / ユーザビリティテスト
```

検証の観点: 木が1〜3か月の技術領域粒度か / 枝が約1週間の学習単元粒度か /
葉が15分〜1日で完了できる具体的な行動(動詞で終わる)か /
ベースライン骨格から逸脱した木が生えていないか。
